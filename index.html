Task 2

Problem Statement

Hiring a lawyer can get pretty expensive in Pakistan. Two non-technical investors thought of an idea to create an AI-based lawyer that can process textual queries and answer them accordingly with citations to the PPC (Pakistan Penal Code) and Constitution. This is especially useful for people living in remote areas and even urban women looking to get a consultation privately. Luckily, you run a software development business and will be providing them services.

Functional Requirements

Functional requirements are the product features or functions developers must implement to enable users to accomplish their tasks.

  1. Authentication of the user whenever they log into the system.
  2. An email should be sent to the user if someone uses their credentials to log in to another device.
  3. A Verification email is sent to the user whenever they register for the first time on the software system.
  4. The system must allow users to reset their password by clicking on "forgot password" and receiving a link to their verified email address.
  5. The application should take string values as the user textual queries.
  6. The application should return answers accordingly by matching keywords in the customer query to the keywords in the database.
  7. The application should also return the citations to PPC and Constitutions with the answers.
  8. The application should not reveal any user information to another user.
  9. The application should ask if the user wants us to save his queries so he can access them anytime.
  10. The application should not save user queries in history if he disagrees since it should be confidential so that women could use it without fear.
  11. The application should ask for user feedback every time the answer is displayed.
  12. The user should be able to view all the requests they have made to the e-lawyer as well as delete them from their account.
  13. They can also edit the contents if they have submitted a wrong request.
  14. A page with a list of the frequently asked topics.
  15. Suggestions based on user keywords.
  16. A page having a list of lawyers available for consultation.
  17. An option to hire any lawyer from the application for the long-term.
  18. An option to review lawyers the user consults.
  19. A help chat option must be there.
  20. An option to delete the user account.

Non-Functional Requirements

Nonfunctional requirements are not related to the system functionality; instead, they define how the system should perform.

  1. User Authentication must be sent within 30 seconds of the login trial.
  2. Login on another device email should be sent within 10 seconds.
  3. Verification email for registration should be sent within 10 seconds.
  4. Password reset link should be sent within 30 seconds.
  5. The first password cannot be reused.
  6. Answers must be displayed within 5 seconds of the query entered.
  7. The list of lawyers should be loaded within 5 seconds.
  8. The application must be scalable enough to handle 500 users.
  9. Application should always be available, even whenever maintenance is ongoing.
  10. The application should be reliable enough to operate failure-free for a specified period in a specific environment.
  11. The system needs to re-establish to recover the data directly affected by the suspended connection.
  12. Monitor the system's health and quickly restore service if it goes into a bad state.
  13. A winning mobile app is entirely compatible with the operating systems it is developed for.
  14. The application should include simple and intuitive navigation and search feature.
  15. Regularly update your app and enrich it with new features.
  16. The design needs to be appealing to the senses. Sophistication and creativity should scream through the screen.
  17. The application should be able to render its layout to different screen sizes.
  18. Privacy of information, the export of restricted technologies, intellectual property rights, etc. should be audited.
  19. The system needs to be cost-effective to maintain.
  20. The system must perform without failure in 95 percent of use cases during a month.
